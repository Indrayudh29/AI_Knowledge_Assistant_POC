Entity Framework Core (EF Core)
Introduction
Entity Framework Core (EF Core) is Microsoft's modern, lightweight, open-source
Object-Relational Mapper (ORM) for .NET applications. It enables developers to
interact with relational databases using .NET objects instead of writing raw SQL
queries for every database operation.
EF Core supports multiple database providers, including:
Entity Framework Core is the recommended data access technology for modern
ASP.NET Core applications. It integrates seamlessly with Dependency Injection,
LINQ, ASP.NET Core, and Minimal APIs.
What is an ORM?
An Object-Relational Mapper (ORM) bridges the gap between object-oriented
programming and relational databases.

Key Components
Entity Framework Core consists of several important components.
Entity
Represents a database table.

public DbSet<Product> Products { get; set; }
Each DbSet corresponds to one database table.

Change Tracking
EF Core automatically tracks changes made to entities.
var product = db.Products.Find(1);
product.Price = 60000;
db.SaveChanges();
EF Core detects the modified property and generates the appropriate SQL update
statement.
Transactions
EF Core supports database transactions.
using var transaction =
db.Database.BeginTransaction();
try
{
db.SaveChanges();
transaction.Commit();
}
catch
{
transaction.Rollback();
}
Transactions ensure data consistency.

Prefer asynchronous methods (ToListAsync, SaveChangesAsync).
Keep DbContext short-lived (Scoped lifetime).

Handle exceptions gracefully.
Profile generated SQL for performance tuning.
Follow the Repository or Service pattern where appropriate.

Real-World Example
Consider an e-commerce application.
Web API
│
ProductController
│
ProductService
│
ProductRepository
│
AppDbContext
│
Entity Framework Core
│
SQL Server
When a client requests GET /api/products, the controller calls the service, which
retrieves data through the repository. The repository uses AppDbContext and LINQ
to query SQL Server. EF Core translates the LINQ expression into SQL, executes it,
maps the results back to C# objects, and returns them to the client. This abstraction
simplifies development while maintaining a clean, layered architecture.
Summary
Entity Framework Core is Microsoft's modern Object-Relational Mapper (ORM) for
.NET applications, providing a productive and maintainable way to interact with
relational databases. By allowing developers to work with strongly typed C# objects
instead of manually writing SQL for common operations, EF Core significantly
reduces development effort while supporting advanced capabilities such as LINQ
queries, change tracking, migrations, relationship management, asynchronous
operations, and multiple database providers. When used with proper architectural
practices, Dependency Injection, and performance optimization techniques, Entity
Framework Core becomes a powerful foundation for building scalable, enterprise-
grade ASP.NET Core applications.
